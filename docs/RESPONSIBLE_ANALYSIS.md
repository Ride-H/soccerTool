# 責任ある解析表現ガイドライン

RPD-X が扱う情報は「公式記録の事実」と「決定論モデルの推定」の2層で構成される。
このガイドラインは、コード・UI・ドキュメント・Issue・対外説明のすべてで両者を
混同させないための規律を定める（Issue #3 / #4 / #5 / #6 / #10 / #13 対応）。

## 1. 事実とモデルの区別（不可侵）

| 層 | 内容 | 表現規則 |
|---|---|---|
| **事実（記録）** | 背番号・氏名・XI・フォーメーション・交代分・得点・警告・会場・主審・観衆 | 出典を明記し断定してよい（テストで固定） |
| **事実（プロバイダ計測）** | 支配率・シュート・xG・パス等のスタッツ | 値と**出典プロバイダをセット**で表示。出典間差を許容する表現にする |
| **モデル推定** | 選手座標・動き・保持チェーン・能力値・危険度・PSY全指標・パスネットワーク・What-if | 「推定」「モデル」「ヒューリスティック」を明示。断定・予測表現を禁止 |

UI 上の常時明示: ステータスバンドの「座標・動き=推定」チップ / インスペクタ「能力値 — モデル推定」/
タイムライン凡例「○記録 ◆SIM生成」/ ツールチップの〔記録〕〔SIM生成〕来歴タグ。

## 2. 禁止する表現・許可する表現

**禁止（例）:**
- 「このシナリオなら勝てた」「〜が起きていたはず」（What-if を予測・反実仮想の真実として語る）
- 「選手Xは集中を切らした」「メンタルが崩壊した」（PSY 指標で実在人物の心理を断定）
- 「危険度がゴールを予測した」（アンカー由来の再現を予測力の証拠として語る）
- 「世界初」「プロレベルの精度」等の検証不能な優位主張

**許可（例）:**
- 「モデル上は、この交代で機会創出が +N% 変化する（決定論的再構成）」
- 「PSY レイヤーの推定では、失点後に覚醒水準の指標が上昇する（イベント駆動ヒューリスティック）」
- 「危険度は較正に用いていない試合でも、記録された全ゴールの直前にCRITICALへ到達した
  （ただし再現アンカーが直前危険度を押し上げる構造は較正試合と共通）」

## 3. What-if（結果再構成）の説明規律

- 常に「予測ではなく決定論的**再構成**」と述べる。
- 実ゴールの再現はアンカー（記録準拠の再現指定）に基づく。よって「なぜ危険度が上がったか」の
  説明はモデル構造の説明であり、**因果の発見ではない**（説明循環の自己認識 — Issue #3）。
- 効果量・係数は較正値（n が小さい）であることを、比較・引用時に添える（Issue #2）。

## 4. PSY レイヤー（心理生理）の特則

- PSY は**生体計測ではない**。位置・イベントから導く決定論ヒューリスティックの解釈支援。
- 実在選手の心理状態・性格・メンタル強度の断定に使わない。個人比較のランキング化をしない。
- 文献は「形状」（逆U字・忘却減衰・HRV低下傾向）のみ借用し、効果量の妥当性は未検証と明示。
- 効果は読み取り専用 — 位置・結果・危険度へフィードバックしない（過剰適合防止 — Issue #10）。

## 5. モデル推定属性（能力値）の特則

- 能力値（pace 等）は表示・空間支配核の半径にのみ使用するモデル推定。
- 推定属性への依存を分離できる「幾何のみモード」を常に維持する（Issue #13）。
- 実在選手の能力の序列として引用しない。

## 6. データ追加時の規律

新しい試合・チーム・選手を追加するときは docs/MATCH_PACKS.md の手順に従い:
1. 事実は複数ソース照合（不一致項目は採用しない/保守的に注記）
2. モデル値はファイル冒頭コメントで「推定」と宣言
3. 共通整合テスト＋試合固有ファクトテストで固定
4. 公開前スキャン（秘密・PII・内部参照ゼロ）を通す

## 7. このガイドラインの適用先

README / UI 文言 / モデル解説モーダル / GitHub Issues・コメント / リリースノート / SNS等の対外発信。
変更時は本ファイルを先に更新し、UI・README の表現を追随させる。
