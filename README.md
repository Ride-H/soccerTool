# RPD-X — 日本 × ブラジル 2026 W杯 R32 / D²-Field 戦術解析ツール

[![CI & Deploy](https://github.com/Ride-H/soccerTool/actions/workflows/ci.yml/badge.svg)](https://github.com/Ride-H/soccerTool/actions/workflows/ci.yml)

**▶ ライブデモ（インストール不要・ブラウザで即起動）: https://ride-h.github.io/soccerTool/**

**2026 FIFA ワールドカップ ラウンド32「ブラジル 2-1 日本」（2026-06-29 NRGスタジアム）** の
検証済み実データをフル収録した、**依存ゼロ・単一HTML**の3D戦術解析ツール。
フル90+9分の3D/VR視点再生、交代・布陣の what-if が**試合結果まで変える**決定論シミュレーション、
距離×人数×時間で危険度を測る **D²-Field v2**、DAW空間プラグイン風の粒子可視化を、
ライブラリを一切使わず素の WebGL2 と JavaScript だけで実装しています。

RPD-1 v2（`rpd1 dashboard.tsx`・前半のみ・2D・サンプル背番号）の後継です。

## D²-Field v2 — 距離 × 人数 × 時間の危険度場

危険度を「距離だけ」でなく **人数** と **時間** も使って合成する6モジュール構造。
既定は **GK除外20人＝チャンス創出度**、トグルで **22人＝得点期待度** に切替。

```
T(x,y)  = exp(−(d_goal/24)^1.7) × シュート角開口（ゴールマウス7.32m）
── 空間・距離 ──
[SDI] 空間支配侵蝕   Σ ctrl·T / Σ T          ctrl = 距離ガウス核の支配率（σ=5.5+pace×3）
[CPR] 保持者圧迫余裕 T(ball) × (1−e^(−d_def²/49)) × 保持度
[PLV] パスレーン開通 max_受け手 [Π(1−e^(−d_lane²/9)) × T(受け手) × 余裕]
── 人数 ──
[OVL] 局所数的優位   σ(攻−守 @ボール16m) + 最終ライン欠員 clamp((2.8−cover)/2.8)
── 時間 ──
[TPA] 持続圧力       Σ_32s 0.85^k · 圧力原値(t−4k)   （波状攻撃で蓄積・HTでリセット）
[TRV] 侵攻速度       clamp(v_ゴール接近/7.5) × T(ball)   （カウンターで急伸）

KIKEN = 100 × clamp( (.18 SDI + .15 CPR + .13 PLV + .22 OVL + .20 TPA + .12 TRV)^0.66 × 1.60 )
        WARNING ≥ 45 / CRITICAL ≥ 75
```

較正は**実試合の3失点すべてが直前に CRITICAL へ到達**するよう決定（52'の決定機セーブ局面も高危険）。

## エンジン保証（テストで担保）

- **純関数 f(t)**: どの時刻を叩いても同一世界（スクラブ・ジャンプ完全対応・順序非依存）
- **速度上限 ≤9.9m/s を構成的に保証**（帯域制限ノイズ＋ガウス窓アンカー＋lerpブレンドのみで合成）
- **ポゼッション・チェーン**: ボールは保持チームの選手の足元に付く。保持者列（ホールド→パス→奪取）を
  実測支配率69/31に整合するよう決定論生成し、金リングで現在の保持者を可視化
- 常時11人×2・キックオフ自陣整列・得点者はシュート地点に到達・交代IN/OUTアニメ
- **交代規則は不可侵レイヤ**: 5人・3窓（HT非カウント）・再入場禁止・GK同士・常時GK1人 —
  手動もAI提案（TacticalSubAdvisor）も布陣エディタも必ずバリデータを通過
- **決定論シナリオ** → 同じ交代・布陣プランは何度実行しても同じ結果（無限に再実行可能）

## 交代・布陣が「試合結果」を変える（What-if Outcome）

交代・フォーメーション・配置の変更は、勝敗そのものを決定論的に再構成します。

- **ゴール消滅**: 得点者/アシスト者がその時刻にピッチ外、または直前の攻撃危険度がシナリオ世界で
  実試合比 55% 未満に低下すると、その実ゴールは消える
- **ゴール追加**: 危険度曲線の増分の積分から期待値 λ を算定し、**決定論ポアソン**で追加得点を発生。
  時刻は増分の極大点、得点者は攻撃値の重み付き決定論選択
- 例: マルティネッリ投入を取り消す → 90+5 の決勝点が消えて **2-1 → 1-1**。
  日本を 4-4-2 化 → 機会増で **2-1 → 2-2** に追いつく（すべて乱数なし・再現可能）

## 使い方

| 操作 | 内容 |
|---|---|
| Space / ←→ | 再生・停止 / ±15秒 |
| 1-5 | カメラ: 放送 / 俯瞰 / ゴール裏 / 追従 / **自由飛行（VR風: WASD+QE+ドラッグ）** |
| H / Z / T / L / G | 危険場（粒子⇄面⇄OFF）/ ゾーン / 軌跡 / 番号 / 20⇄22人 |
| 選手クリック | 選択+インスペクタ（走行距離・疲労・脅威寄与・能力値） |
| ゾーニング視点 | 中立 / 日本目線 / ブラジル目線 — 各選手の支配空間を粒子で可視化（白い継ぎ目=ゾーン境界） |
| 配置エディタ（ミニピッチ） | 陣形変更・選手ドラッグで微調整・クリック2回で入替・選択中にベンチ選手で先発差替 |
| 交代ビルダー | ピッチ上→OUT・ベンチ→IN を自動セット → 分指定 → 追加。AI提案は全て規則検証済み |
| タイムライン | ドラッグで頭出し・イベントピン・実試合 vs シナリオ破線比較 |

URLパラメータ: `?t=3510&cam=tactical&zone=JPN&field=particles&play=0&sel=BRA:5&demo=blitz`
（頭出し / カメラ / ゾーン視点 / 危険場モード / 停止 / 選手選択 / シナリオデモ`sim`|`blitz`）

## 別の試合で使う（過去データ不要）

「カスタム試合」→ JSONテンプレート編集 → 読み込み。
選手情報（背番号・氏名・ポジション、任意で能力値/XI/イベント）だけで、
チーム強度からポゼッション・得点・危険度を決定論生成し、同じ3D/交代シム/D²-Fieldが動きます。
API: `RPDX.generic.createMatch(cfg)`（`rpdx/src/generic.mjs`）。

## 構成

```
rpdx/src/    noise / formations / data_match(検証済データ) / engine / danger / subs / sim / generic
rpdx/app/    render3d(自作WebGL2・粒子/半透明) / ui / app.css / index.template.html
rpdx/test/   54テスト（engine / danger / data / subs / sim / lineup / generic）
dist/        rpdx.html（配布用単一ファイル）/ rpdx_artifact.html（claude.ai Artifact用）
```

すべて `globalThis.RPDX` 名前空間方式（import/export不使用）で、`build.mjs` が連結して単一HTMLに焼き込みます。
