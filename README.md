## D²-Field v2 — 距離 × 人数 × 時間の危険度場

危険度を「距離だけ」でなく **人数** と **時間** も使って合成する6モジュール構造。
既定は **GK除外20人＝チャンス創出度**、トグルで **22人＝得点期待度** に切替。

```
T(x,y)  = exp(−(d_goal/24)^1.7) × シュート角開口（ゴールマウス7.32m）
── 空間・距離 ──
[SDI] 空間支配侵蝕   Σ ctrl·T / Σ T          ctrl = 距離ガウス核の支配率（σ=5.5+pace×3）
[CPR] 保持者圧迫余裕 T(ball) × (1−e^(−d_def²/49)) × 保持度
[PLV] パスレーン開通 max_受け手 [Π(1−e^(−d_lane²/9)) × T(受け手) × 余裕]
── 人数 ──
[OVL] 局所数的優位   σ(攻−守 @ボール16m) + 最終ライン欠員 clamp((2.8−cover)/2.8)
── 時間 ──
[TPA] 持続圧力       Σ_32s 0.85^k · 圧力原値(t−4k)   （波状攻撃で蓄積・HTでリセット）
[TRV] 侵攻速度       clamp(v_ゴール接近/7.5) × T(ball)   （カウンターで急伸）

KIKEN = 100 × clamp( (.18 SDI + .15 CPR + .13 PLV + .22 OVL + .20 TPA + .12 TRV)^0.66 × 1.60 )
        WARNING ≥ 45 / CRITICAL ≥ 75
```

較正は**実試合の3失点すべてが直前に CRITICAL へ到達**するよう決定（52'の決定機セーブ局面も高危険）。

## PSY レイヤー v1 — 心理生理（テニス解析アルゴリズムの移植・非予測）

テニス解析で確立された手法をサッカーへ移植した**読み取り専用の解釈レイヤー**。
位置・イベント・結果には一切影響せず、決定論（スクラブ完全一致）を保ちます。

```
モメンタム M(t) = Σ_ev w(ev)·±·e^(−Δt/300s)      忘却係数付きイベント連鎖（差分方程式の連続化）
覚醒 AR        = 基線 + Σ活性化インパルス·(1−0.4·MF) + 接戦×終盤 + 現在危険度 − 12·MF
                 （得点も失点も警告も「活性化」を上げる — valence ではなく activation）
精神疲労 MF    = time-on-task + 被危険度曝露の減衰積分(τ25分) + 0.22·身体疲労 + 文脈(警告/失点/ビハインド)
                 （HTで部分回復）
集中力 CN      = 100·e^(−(AR−60)²/2σ²)·(1−0.45·MF)·(1−0.15·身体疲労)    Yerkes-Dodson 逆U字
自律神経       = 交感⇄副交感バランス + HRVプロキシ（安静基準比%・精神疲労で反応性低下）
```

- UI: PSYパネル（チームモメンタム+スパークライン・出場11人平均）、インスペクタ（集中力・覚醒・精神疲労・HRV）、選択選手の覚醒オーラ（青=低覚醒/金=至適/赤=過覚醒）
- **重要**: 生体計測ではなく決定論ヒューリスティックによる解釈支援です。心理状態の断定・予測ではありません。
  効果量は文献の定性形状のみ借用し、実試合への適合最適化はしていません（詳細はUI内「モデル」）
- アルゼンチン×エジプトの**2点差逆転劇**でモメンタム大反転・終盤の高覚醒/高疲労が観察できます（テストで性質を固定）

## 放送級オーバーレイ・人型選手描画

- **速度ラベル**: 選択選手に「背番号 | 名前 + km/h」、20km/h以上の疾走選手に自動表示（決定論ヒステリシス）
- **点線ラントレイル**: 選択選手の走路を等間隔ドットで（放送グラフィクス準拠）
- **パスライン**: フライト中の出し手→受け手を点線表示（**パスカットは赤系**・受け手にリング）
- **ボール光跡**: 加算グローのコメット
- **危険ホットゾーン**: WARNING以上のとき脅威極大エリアを半透明矩形で強調
- **人型選手（#154/#155・スキンドメッシュ）**: 全身を **1枚の連続メッシュ**（プロシージャル生成・15ボーンの
  Linear Blend Skinning）で描画。膝・肘・股関節・脊椎で表面が連続的に曲がり、関節球で継ぎ目を隠す
  「風船人形」を脱却。背番号は胸ボーンに追従。既存のゲイト（ケイデンス/膝屈曲/前傾/胴ひねり）は
  ボーン回転として流用し挙動は不変、審判も同じ人型経路。旧カプセル描画は移行期の切り戻し用に
  `?fig=capsule` で温存（描画層のみ・golden不変）。
- **造形・シルエット（#155）**: 楕円断面で **V字テーパー（肩幅>腰幅）**・胸郭の膨らみ・腹部の絞り・
  平たい背中。**僧帽筋スロープで首と肩を連続**（頭が"刺さっている"感を解消）。腕は**ミトンの手**で終端
  （筒で終わらない）、脚は**つま先/踵のあるブーツ**。頭部は顎と髪のジオメトリで人体シルエット。
  **選手ごとの決定論的な体格差**（身長・横幅を選手キー由来で小幅にバリエーション＝全員同一体型を回避）。
- **アニメーション表現力（#156）**: **2ボーン解析フットIK**（ポールベクトルで膝を前方へ・膝反転なし）＋
  速度から算出したストライドで**支持脚を接地点にロック**（歩行速度でスケーティング解消・走行速度は
  脚長×自然なケイデンスの物理限界で残存を低減）。**重心スウェイと呼吸**（アイドルでも静止しない）、
  加速度（ローパス）駆動の**前傾アンティシペーション**、頭・首でボールへ**注視**（角度クランプで分配）、
  シュート/パス時の**キックスイング**（ボールのフライト開始＝時刻の関数で決定論・蹴り足は選手ごと）。
  すべて時刻/位相の関数でスクラブ安全。※mocap 級の動きの質はモーション資産依存として対象外（正直な限界）。
- **セルフシャドウ＋接触AO（#157・VIS-04）**: **頂点ベイクAO**（腋/股/顎下/内側を暗く＝一体感・セルフ遮蔽の近似）を
  スキンドメッシュに焼き込み全 tier で常時適用。**Cinematic tier では方向性シャドウマップ**
  （深度FBO＋PCF・選手プロキシが芝に人型のキャストシャドウを落とす）を `RPDX.quality.flags.shadowMap` 経由で有効化
  （**VIS-00 の品質ティアの初の実消費**）。Lightweight tier は従来の円盤影＋頂点AO。**二重影は回避**（影マップ有効時は円盤影を抑制）。
  深度テクスチャ非対応環境は円盤影へ安全フォールバック。※真のセルフシャドウ（腕→胴等）は頂点AOで近似（プロキシ影の破綻回避・正直な限界）。
- **材質差別化（#158・VIS-03）**: `FS_SKIN` を色ID別のシェーディングに分岐＝**肌・ジャージ・革が別材質に見える**。

  革（ブーツ）＝鋭い反射、髪＝弱い光沢。Blinn-Phong の強度/鋭さを材質別に。過剰なスペキュラは避け控えめに。

- **描画リアル感 v1（#134・描画層のみ・結果/再現性は不変）**: 選手の**接地ソフト影**（空中戦では縮小/減衰）、

## 視覚回帰ゲート（#153・VIS-08）

描画（`render3d.mjs`）の破綻を CI で検知する**視覚スモーク** `rpdx/test/visual/smoke.mjs`。
外部パッケージ不使用 — **Node 組込み WebSocket + CDP 生プロトコル**で Chrome（CI はランナー同梱）を駆動する。

- **決定論の土台**: 描画側乱数のシード化（ゲイト位相=選手キー由来・芝/観客テクスチャ=固定シード・
  `Math.random` 全廃を契約テストで走査）＋ 仮想時間駆動（rAF=16.6ms刻み）＋ ショットモード開始時の
  カメラ即時スナップ。同一URLの再実行でピクセル差 0.00〜0.02% を実測。
- **存在チェック主・golden 補助**: 緑ピッチ/芝縞コントラスト/接地影（相対暗色ブロブ）/
  背番号の描画寄与（トグルOFF起動との差分）をアサートし、golden 画像（3シナリオ: 俯瞰・放送・追従）
  とは**許容差つき**（チャネル差>20 が 2.5% 以内）で比較 — ソフトウェアレンダラの環境差を吸収しつつ
  メッシュ割れ級の大破綻を検知する。
- **golden の更新は意図的な視覚変更の PR のみ**（`UPDATE_GOLDEN=1` で再生成・レビュー承認必須 —
  CONTRIBUTING 参照）。失敗時は CI が実画像（`out/`）をアーティファクトとして保存する。

## 戦術フェーズ自動分類（#32）

各時刻の局面を **set-piece / transition / build-up / progression / finishing** に決定論分類
（保持チーム・前進度・奪取直後・リスタートからの決定木 — ヒューリスティック明示・非予測）。
タイムライン上端の**フェーズ帯**で試合の構造が一目で分かり、チーム別のフェーズ配分・
被プレス時間も集計（支配が明確な試合では「劣勢側=ビルドアップ/トランジション比が高い」が成立・拮抗試合（49/51）は健全域チェックに場合分け — テスト固定）。
API: `RPDX.tactics.phaseAt / phaseShares / phaseStrip`

**実効フォーメーション & 形メトリクス（#33）**: 宣言陣形ではなく「いま実際にどう並んでいるか」を中立に測定。
凸包面積（コンパクトネス）・幅/縦・重心・実効ライン構成（深さギャップで3ライン分割）・ライン間距離・
Voronoi占有率（4m格子近似）を各時刻で決定論算出。試合情報モーダルに現在時刻のスナップショット表を追加
（守備側の日本は低ブロックで凸包 約827m²・ブラジルは攻撃時 約2146m²、というコンパクト差がモデル上で成立）。
API: `RPDX.tactics.shapeMetrics / voronoiShare`（読み取り専用・位置と結果に不干渉）。

**守備構造解析 v1（#117）**: 攻撃側視点だけでなく**守備の構造**を読み出す2段構成（読み取り専用・golden不変）。
- **守備側帰属ビュー**（`T.defenseProfile`）: 相手の危険度モジュール平均を「守備側が何を許したか」として再集計
  （被CPR・被PLV・被OVL・被TRV・被KIKEN）。例: FRA-ESP ではスペインの被TRV 1.08 vs フランス 2.08
  ＝カウンターを半分に抑えた「レスト・ディフェンスの実体」が数値化される。
- **守備ブロック読み出し**（`T.defenseBlock / defenseBlockAt`）: **非保持時のみ・GK除外・ボール相対**で
  ライン高さ・ブロック幅×縦・スライド追従差・**中央レーン閉鎖度**（PLVレーン式の逆読み・def編集=#106が効く）を算出。
  例: BRA-JPN では日本の低ブロック（ライン高 14.9m・縦 36m）とブラジル（30.6m・60m）が明確に判別される。
- UI: 試合情報モーダル「守備構造 v1」（オンデマンド計算・シナリオ追従）。実測ではなくモデル推定上の観察と明示。

## 相手分析体制の脆弱性プロファイラ（#59・opponent v1）

「リアルタイム解析→ベンチ→選手」という**相手のフィードバック体制そのもの**の脆弱性
（HT15分の意思決定）を、体制パラメータ（人数・伝達段数・ツール/属人依存）から決定論評価。
3類型アーキタイプ（人海戦術型/AI・テック特化型/現場主義型）のHT時間配分（収集/会議/共有）と
4軸スコア（情報遅延・意思決定ブレ・システム依存・**HT修正力の脆弱性**）を試合情報パネルに表示。
**実在の連盟・チームへの断定ではなく**、宣言された仮定のモデル評価（パック未宣言なら実チームに帰属しない）。
さらに**リアルタイム意思決定負荷**（#60）: 試合の情報フロー圧 IFL(t)（イベント密度×危険度変動×
局面切替）と体制の処理能力から**前半の処理飽和度**・HT持ち込み情報量（backlog）・実質共有時間を算出
（人海戦術型が最も飽和 — テスト固定）。選手側の**認知キャパ**（HT変更点数>3で過負荷）も判定。
**HT修正力シミュレーション**（#61）: シナリオに `opponentHt: {team, archetype}` を指定すると、
その相手のHT近傍の布陣修正が体制脆弱性に応じて**遅延（総合スコア+backlogで0〜10分）・鈍化**
（浸透45〜135秒）する — 「意思決定の弱い相手」への後半プランを結果再構成つきで比較可能。
既定OFF＝現行挙動と完全一致（ゴールデンマスターで保証）・世界シード（保持チェーン）は不変。
**カウンタープラン提案**（#62）: 弱点タイプ→戦術テンプレ（二段可変=集計リセット / HT直前可変=編集時間の剥奪 /
前線入替=ミスマッチ / 非定型立ち上がり=自動分類外し）を**実行可能な what-if シナリオ**として決定論生成し、
相手の情報環境への影響（Δ飽和・Δbacklog・Δ実質共有）と自軍の結果再構成を actual 比で数値比較
（`counterPlans / evaluatePlans` — 断定ではなくモデル上の比較）。
API: `RPDX.opponent.profile / htBudget / ARCHETYPES / setupOf / iflAt / htSaturation / htCognitive / counterPlans / evaluatePlans`・`E.htCorrectionOf`

## エンジン保証（テストで担保）

- **純関数 f(t)**: どの時刻を叩いても同一世界（スクラブ・ジャンプ完全対応・順序非依存）
- **速度上限 ≤9.9m/s を構成的に保証**（帯域制限ノイズ＋ガウス窓アンカー＋lerpブレンドのみで合成）
- **ポゼッション・チェーン v4**: ボールは保持チームの選手の足元に付く（アンカー再現中も
  近傍なら足元へ吸着 — ドリブルが途切れない）。保持者列は持続性マルコフ
  （定常分布=実測支配率・連続保持は優勢~7本/劣勢~3.4本の実試合水準）で決定論生成。
  奪取は前保持者近傍のタックル/インターセプト（接触収束アンカー）、**スローイン/コーナー/
  ゴールキック**を決定論挿入（~60回/試合・「静止→投げる」で次のフライトがスロー/クロスになる）。
  記録イベント直前は保持チームを記録へ拘束し、パックは**保持台本 `chainForce`**（区間のチーム/
  選手指定 — パスカット等の実況ストーリー）を持てる
- **セットピース挙動**: コーナーは攻4守5がボックスへ密集、攻撃ゴール38m内のFKは守備が
  9.15mの壁を形成し被ファウル側が再開、**得点後は全員の帰陣を待ってキックオフ**（台形集合窓・
  **ボールはセンタースポットに静止し失点側が保持して味方へ蹴り出す**・ピン中は相手が寄せない）
- **集団サージ**: ポゼッション基調波の微分で、カウンター時は全体が一斉撤退／攻勢転換時は
  全体が押し上がる（役割別スケール・速度上限内）
- **階層プレッシング & トリガ**（#29）: 最近接プレッサーは0.9mまで密着、2番手がカバー（オープン
  プレー保持者への最近接守備者は中央値2〜6mをテストで固定）。相手が自陣~40%でビルドアップすると
  **協調プレスが点灯**（連続level・`pressTriggerAt`）— 1st増圧・2ndは**カバーシャドウ**（保持者→
  支援重心のパスレーン上）・3rdも連動
- **スタミナ→行動フィードバック**（#44）: 疲労（在場時間×スタミナ属性・交代でリセット）が
  ブロック追従・集団サージ・プレス強度・オフボールランを減衰 — フル出場選手は終盤に平均速度が
  約25%低下し（テスト固定）、フレッシュな交代選手との質的差が出る
- **意図的オフボールラン**（#30）: 攻勢×前進局面で FB/WB が**オーバーラップ**（ボールサイドを
  追い越して幅を取る）、ST/W/AM が**裏抜け**（オフサイド境界の手前まで・周期パルスで出入り）。
  記録イベント窓では停止（較正済み再現局面を保護）・チェーン選定にも不干渉（表示上の走り）
- **相互分離**（#28）: 社会力モデルの斥力項をエンジン座標に導入 — 選手同士のランダムな重なり
  （すり抜け）を解消（最小ペア距離~0.3m保証・プレス密着や接触収束など意図的な近接は保存）
- **協調ラインコントロール & オフサイド**（#27）: 最終ライン（CB/FB/WB）を合意 x へ同期させ
  「1枚のラインとして」上下動（守勢で自陣へ・攻勢で押し上げ）。オフサイド境界/判定
  （2nd-last守備者×ボールの規則・読み取り専用 `offsideLineAt`/`isOffsidePos`/`offsideAttackersAt`）
- **GK守備幾何**（#31）: ボール→ゴールの二等分線上に立つ角度圧縮＋自陣深部でのスイーパー飛び出し。
  純関数（平滑ボール依存）で速度上限を保ちつつ、至近の脅威ほど前へ出て角を狭める
- **ボール物理**（#46/#50）: 決定論のバウンド（反発係数の幾何ホップ）＋マグヌス曲がり（飛球の横カーブ・
  端点保存・±1.4m）。危険度の x,y 較正は不変。軌道は**チェーン境界でもC0連続**（フライト所要時間に
  速度上限由来の下限・リスタート解放はセグメント内に完了 — 瞬間移動なし・テストで固定）
- **選手アンカーの非対称窓**（sigmaL/sigmaR）: 到達は鋭く・解放は緩やかに — 独走の加速
  プロファイルとシュート後の減速を両立
- 常時11人×2・キックオフ自陣整列・得点者はシュート地点に到達・交代IN/OUTアニメ
- **交代規則は不可侵レイヤ**: 5人・3窓（HT非カウント）・再入場禁止・GK同士・常時GK1人 —
  手動もAI提案（TacticalSubAdvisor）も布陣エディタも必ずバリデータを通過
- **決定論シナリオ** → 同じ交代・布陣プランは何度実行しても同じ結果（無限に再実行可能）
- **プロパティベース・ハーネス**（#36）: シード付き生成器で交代シナリオ空間（2^n部分集合）を

- **専門知ルール・オラクル**（#37）: 競技規則・定性知識のルール束（常時11人×2/GK各1・ボール場内・

  actual と what-if の両世界でサンプル検証 — 実測なしでも「サッカーとして破綻していない」ことを常時保証
- **ゴールデンマスター回帰**（#35）: 世界状態（座標・ボール・保持者・結果再構成）のハッシュを

## 編集/静止フレームの方向的タクティカル解析（#85）

編集フレームや任意の静止フレームに `RPDX.tactics.frameAnalysis(match, state)` を掛けると、その瞬間の
**位置・構造の帰結**を返す（読み取り専用・決定論）: 危険度ホットスポット（`fieldAt`）・実効フォーメーション
（`shapeFromState` #33）・Voronoi占有・オフサイド境界（`frameOffside` #27）・ボール周辺の数的過多、
そして**位置系の対策トップ3**（局所同数化 / 背後スペースへのラン / 高危険地点のカバー / 逆サイドの幅取り）。
編集モードのバーに解析が表示され、選手を動かすたびに更新される。**テンポ/因果は断定しない**
（1フレームは空間・TPA/TRVは時間依存で対象外）＝「支援であって指示ではない」。

## 編集フレームからの再合成（#83・「編集後の動き」）

編集バーの「この配置から再生」で、編集位置を**制約（scenario級の editAnchors）**に翻訳し、
**以降を既存エンジンで再合成**する（`RPDX.scenlib.scenarioFromFrame`）。編集時刻でその配置を通過し、
その後は各選手が役割・保持チェーン・プレスへ**もっともらしく続く**。アンカーσは編集前位置からの
移動距離に比例（速度上限 ≤9.9m/s を構成的に保証）。編集は scenario 級のため **actual/既定world・golden は不変**。

> **正直な限界**: これは「編集と整合する妥当な再合成」で、実座標からの厳密な因果的続きではない。
> 編集の影響は数十秒でベースラインへ減衰する（1フレームを初期条件とする過渡）。厳密な創発は #84（別トラック）。
> ボール保持の再合成は将来拡張（残）。

## 選手能力値・名前の編集（#89）

インスペクタの「選手を編集」から、能力値（速度/持久/守備/攻撃/技術/空中）と名前を編集できる。
編集は **scenario 級の上書き**（`RPDX.engine.attrsOf` / `nameOverrideOf`）で、**actual/既定world・golden は不変**。
**pac は危険度の空間支配核 σ に効く**ため、編集すると危険度場・結果再構成が変わる（入力次第で結果が動く）。
名前はコスメティック（位置・危険度に無影響）。**位置（動き）は能力値では変わらない**（軌跡は陣形＋ノイズ＋速度上限）
＝能力値は危険度の重みを変える。
**att/tec は攻撃支配、def は守備支配**として危険度の空間支配（SDI/ctrl）に係数化される（#90）。
較正保護（不可侵）: 係数は**上書き前からの差分ベースで未編集=1.0** ＝ 収録実試合の危険度・OOS・golden は完全不変。効果は有界（±約24%）・ソフト。
**v2（#106）**: att/tec は **CPR（保持者の脅威）・PLV（受け手の脅威）** にも、def は **CPR の実効圧迫距離・PLV のレーン・シャドウ幅**にも係数化。
さらに **GK の def/aer** が相手チームの危険度全体を抑制/増幅（±約16%有界）＝ **GK を編集すると結果が動く**。
いずれも差分ベース（未編集=1.0）で全収録試合の total ビット一致をテスト固定。

### シナリオの JSON 往復（取込→反映→書出・#91）

シナリオ（交代・布陣・能力値/名前の上書き・編集フレーム）を**統合スキーマの JSON で往復**できる（`RPDX.scenlib.serializeBundle` / `parseBundle`）。
取込は**ファイル選択・ドラッグ&ドロップ・`?data=`**、書出はダウンロード、任意で**端末保存（localStorage）**。
**すべて端末内で完結し、外部送信・サーバ/リポジトリへの蓄積は一切ない**（依存ゼロ・オフライン）。
`roster` 形式（選手行の能力値/名前）も受け付け、**能力値は値域にクランプ**・**未指定は既定（squad 由来）のまま**。
取込は**シナリオ級の上書きとしてのみ反映**され、収録試合（match/golden）は不変。不正 JSON は安全に拒否。

### 多重編集の記憶（#123）

停止フレーム編集は**何回でも重ねられる**: 「編集→この配置から再生」を繰り返しても前回の編集は消えず、
**別時刻の編集は共存・同一選手×近接時刻（±25s）の再編集だけ最新で置換**される（10回編集の通過保証をテスト固定）。
編集アンカーは記録イベント再現やプレス/ラン等の**全ての変位より最後に適用**され、編集した座標をその時刻で厳密に通過する。
**ボールも再合成対象（#127）**: 編集モードで動かしたボールは `editBall` として蓄積され、その時刻を通過（10回規模・#50 の速度/連続性維持）。
**保持者の編集も厳密通過**し、ボールが足元に追随する（吸着強制・10回規模でテスト固定）。チェーンは編集を再シードしない（同一ハッシュ⇒同一保持者系列の契約）。
交代リストに「✏ 配置編集 @分（n人＋ボール）」の履歴行が並び、**✕ で該当時刻の編集だけ取り消し**できる。JSON 往復（#91）も全量対応。

### 手動スコア修正 — 追加点と減点（#128）

交代パネルの**「スコア修正（what-if）」**から、任意チーム・分で**得点の記帳（＋追加）**と**ゴール取消（−減点）**ができる。
- 追加は #80 の機構を拡張した kind「手動（記帳）」（PSY への心理増幅は中立）。外的6種（誤審PK等・「仮定」明示）も選択可。
- 減点は指定分±60秒の**記録/生成ゴールを取消**（理由「手動取消（what-if）」を明示・再現アンカー/祝祭も抑制・**実記録は不変**）。
- 各チーム 追加5件・減点5件まで。リスト行の ✕ で戻せる。JSON 往復対応・未指定世界はビット不変（golden）。

### 編集可能なテンプレート試合（実測なしの起点・#92）

**HTML を開いた既定はこの「テンプレート試合」**（`RPDX.generic.templateMatch()`・`generic.createMatch` の薄い活用）。
収録2試合（較正済み実データ・「ブラジル×日本」「アルゼンチン×エジプト」）は**試合スイッチャまたは `?match=<id>`** で選べる。
テンプレの既定は**チームA／チームB**（選手名 `プレイヤーA1..`／`プレイヤーB1..`）で、**収録試合と同一 API**（エンジン・危険度・交代・編集・JSON 往復）で完全動作し、
そこから能力値・名前・**背番号**・配置を編集して自チーム構成へ寄せられる。
**正直な表示**: テンプレ（および任意のカスタム試合）は `meta.calibrated === false` で、UI に**「未較正・汎用推定」バッジ**を常時表示する
（危険度・支配率は較正済みの実試合値ではない）。**テンプレはレジストリに常時登録せずオンデマンド生成**するため、収録2試合の golden／テストは不変。

**左リストからの背番号・名前編集（#92b・未較正試合のみ）**: ロスター行の ✎ から**背番号と名前を直接編集**できる（`RPDX.generic.editEntry`）。
背番号は選手の識別キーのため、変更時は **XI 割当・主将順・交代・能力値/名前の上書き参照を一括で再マップ**し、重複/範囲（1〜99）を検証する。
**ポジション（GK/DF/MF/FW）も編集可（#92d）**: DF/MF/FW 間は自由。**GK は各チーム1人の不変式**を保ち、ある選手を GK にすると旧 GK を自動で DF に降格し XI の GK スロットをスワップ、唯一の GK を外野化しようとした場合は阻止する。
**収録実試合（較正済み）は公式記録・golden 保護のため編集不可**（名前はインスペクタのシナリオ級上書きで可）。編集はすべて端末内・送信なし。

**カスタム試合＝テンプレと同一定義**: 「カスタム試合」モーダルの初期 JSON は既定テンプレ（`RPDX.generic.template()`）と**同一定義を共有**し、`templateMatch()=createMatch(template())`。
**出典表示の整合（#92c）**: 試合情報モーダルの出典・「公式記録」表記は**較正状態とシナリオ状態に追従**する
— 未較正試合（テンプレ/カスタム）は「モデル生成・未較正／出典なし」を明示し、what-if シナリオ表示中は「記録・スコア・出典は収録実試合のもの（シナリオ結果ではない）」と注記する（実記録の断定を出さない）。

## 数的不利 — 10 vs 11 のシミュレーション（#81）

選手インスペクタの「🟥 退場」から、**その時刻に退場（レッド/交代枠なし負傷）を発生させる what-if** を作れる。
- **エンジン**: `scenario.outages`（シナリオ級・v1はチーム毎1件・非GK）。退場後はそのチームが**10人**になり、
  役割適合の貪欲再割当で **10人シェイプ（4-4-1 / 4-3-2 / 5-3-1・自動または指定）** へ決定論リシェイプ。
  保持は相手側へシフト（2分ランプ）し、**支配率・危険度・結果再構成が方向的に変わる**
  （例: 60'退場で支配率 71.6→64.3%・相手危険度 +57%）。
- **規則検証**: 退場者の以降の交代OUTは自動取り下げ・GK退場は v1 非対応・在場外の指定は拒否。
- **UI**: HUD に「10 vs 11」バッジ・交代リストに退場行（✕で取り消し）・JSON往復（#91）にも対応。
- **golden安全**: `outages` 未指定の世界は位置・ハッシュともビット一致（テスト固定）。

## 外的失点の注入と巻き返しビルダー（#80）

**危険度に依存しない仮説ゴール**（誤審PK・オフサイド見逃し・デフレクション・GKミス・セットピース混戦・オウンゴール）を
what-if として注入できる（`scenario.shockGoals`・交代パネルの「外的失点（仮定）」フォーム・チーム毎2件まで）。
- **責任ある表現（不可侵）**: これは仮説であり、**実試合・実審判にその事象があったという断定ではない**
  （収録記録は不変・得点者個人は特定しない=no:null）。UI・イベントに「仮定」を常時明示。
- PSY への接続: 理不尽な失点ほどモメンタム/覚醒のスイングが大きい（kind別重み・テスト固定）。
- **巻き返しビルダー**: ビハインド側を特定し、**最終ビハインド区間の開始**から介入する回復プラン
  （攻撃的シフト/攻撃的交代/複合）を決定論生成し、目的関数（Δ得点差・Δ攻撃危険度・被リスク）で降順ランク
  （`RPDX.scenlib.recoveryPlans`・結果パネルの「巻き返し案を比較」・**効果が負の案も正直に表示**）。
- golden安全: `shockGoals` 未指定の世界・結果は従来と完全一致（テスト固定）。

## 交代・布陣が「試合結果」を変える（What-if Outcome）

交代・フォーメーション・配置の変更は、勝敗そのものを決定論的に再構成します。

- **ゴール消滅**: 得点者/アシスト者がその時刻にピッチ外、または直前の攻撃危険度がシナリオ世界で
  実試合比 55% 未満に低下すると、その実ゴールは消える
- **ゴール追加**: 危険度曲線の増分の積分から期待値 λ を算定し、**決定論ポアソン**で追加得点を発生。
  時刻は増分の極大点、得点者は攻撃値の重み付き決定論選択
- 例: マルティネッリ投入を取り消す → 90+5 の決勝点が消えて **2-1 → 1-1**。
  日本を 4-4-2 化 → 機会増で **2-1 → 2-2** に追いつく（すべて乱数なし・再現可能）

## 別の試合で使う — マッチパック（検証済み実データ）とカスタム試合

**マッチパック（実データ）**: `rpdx/src/data_match*.mjs` を置くだけでビルドに自動同梱され、

| 試合ID | 試合 | 出典 |
|---|---|---|
| `wc2026-r32-bra-jpn`（既定） | R32 ブラジル 2-1 日本 | FIFA公式記録・ESPN 760487 ほか |

## 性能（#40 ベンチ & 予算）

`node rpdx/tools/bench.mjs` が代表操作（チェーン構築・再生ループ・危険度曲線・結果再構成・
生理集計・走行距離）を決定論ワークロードで計測。CI は主要4操作に**性能予算**を課し
（並列実行の競合を見込んだ余裕設定・桁違いの退行のみ検出）、最適化の効果と退行を数値で管理します。
**#38 Worker オフロード**: 危険度曲線の計算を **Blob Worker**（core scriptタグ＝DOM非依存の
計算層をそのまま再評価・依存ゼロ維持）へ移し、メインスレッドのヒッチを排除。シナリオは
直列化（#34）で受け渡し＝内容ベースhashで同一世界・同一結果。Worker不可・無応答2.5秒で
従来のチャンク計算へ自動フォールバック（二重完了ガードつき・結果は決定論でどちらも同一）。
**#39 高速化**: 基礎位置の per-t メモ（同一時刻の重複評価 ~38→~23回）＋相互分離の空間ハッシュ
（加算順序保存）で、走行距離 −18%・生理集計 −32%・総計 −18% — **出力はビット同一**
（ゴールデンマスターが保証・最適化中の値混入バグも2度検出した実績）。
